## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Radio indicator-to-label internal gaps only if a custom radio control is required |
| sm | 8px | GroupBox internal padding; vertical gap between radio choices; rounded replacement for Tk `padding=5`, `padx=5`, `pady=5` |
| md | 16px | Default element spacing; gap between Settings groups |
| lg | 24px | Settings tab page padding; matches existing `ui/settings_tab.slint` placeholder padding |
| xl | 32px | Reserved for larger cross-tab layout gaps; do not introduce inside the two Settings groups |
| 2xl | 48px | Reserved for major section breaks; not used in this phase |
| 3xl | 64px | Reserved for page-level spacing; not used in this phase |

Exceptions: none. Reference Tkinter 5px frame padding/margins are intentionally approximated as 8px to keep this Slint phase on the project 4px spacing grid while preserving the compact desktop feel.

Layout contract:
- Replace the current centered placeholder with a top-aligned Settings content area.
- Use one horizontal row containing exactly two titled option groups, in this order: `Update Channel`, then `Log Level`.
- Each group uses vertical option stacking with left-aligned radio labels.
- Keep group widths content-driven; do not stretch radio labels across the whole tab unless required by Slint layout constraints.

## Typography

Use exactly these sizes and no additional weights in this phase:

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 14px | 400 | 1.5 |
| Label | 14px | 600 | 1.4 |
| Heading | 16px | 600 | 1.2 |
| Display | 18px | 600 | 1.2 |

Typography contract:
- Radio option text uses Body: 14px / 400.
- Group titles `Update Channel` and `Log Level` use Label or the active Slint `GroupBox` title style if it maps to 14px / 600.
- The Settings tab should not add a large visible `Settings` heading unless the surrounding `TabWidget` title is insufficient for accessibility; if retained, use Display: 18px / 600 and place it above the two groups with `md` spacing.
- Tooltips and status text use Body: 14px / 400.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #F3F3F3 | Settings tab background and inactive page surface |
| Secondary (30%) | #FFFFFF | Option group interior surfaces / card-like group bodies |
| Accent (10%) | #0067C0 | Selected radio indicator, keyboard focus ring on the active radio option, optional successful save status only |
| Destructive | #C42B1C | Save/load error text only; no destructive actions exist in this phase |

Accent reserved for: selected Update Channel radio, selected Log Level radio, keyboard focus ring for the currently focused radio option, and an optional transient successful-save status indicator. Do not use accent for every label, group border, or static heading.

Color contract:
- Preserve the conservative native desktop look from the reference Tkinter UI; do not introduce a new branded theme.
- If the active Slint style supplies native platform colors for `GroupBox` and radio controls, prefer those native colors while preserving the 60/30/10 emphasis contract above.
- Error states must use the Destructive color and plain text; do not display modal errors for silent reference-compatible settings repair.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | None visible — settings save immediately when a radio option is selected |
| Empty state heading | Not shown — Settings always renders `Update Channel` and `Log Level` groups |
| Empty state body | Not shown — if settings are missing or invalid, load repaired defaults before rendering the tab |
| Error state | `Could not save settings. Your previous setting was restored.` |
| Destructive confirmation | None — this phase has no destructive actions |

Reference-visible labels must match exactly:

| Group | Order | Label | Persisted value |
|-------|-------|-------|-----------------|
| Update Channel | 1 | `All: GitHub & Nexus Mods` | `both` |
| Update Channel | 2 | `Early: GitHub` | `github` |
| Update Channel | 3 | `Stable: Nexus Mods` | `nexus` |
| Update Channel | 4 | `Never: Don't Check` | `none` |
| Log Level | 1 | `Debug` | `DEBUG` |
| Log Level | 2 | `Info` | `INFO` |
| Log Level | 3 | `Error` | `ERROR` |

Tooltip copy, if implemented in this phase, must match the reference constants:
- Update Channel: `GitHub will always have the latest release. Nexus Mods releases may be delayed due to their review process or to await more testing.`
- Log Level: `Sets the minimum importance level for messages sent to the log file. DEBUG: Extra verbose. INFO: Default. ERROR: Only log errors.`

Validation and repair copy:
- Malformed JSON reset, partially invalid value repair, unknown-key cleanup, and `download-source.txt` fallback are logged only and should not show user-facing warning copy in the Settings tab.
- Save failure is the only Settings-tab-visible error state for Phase 2; after showing the error, restore the last persisted radio selection.

## Interaction Contract

Source behavior: `CMT/src/tabs/_settings.py` creates `ttk.Radiobutton` controls and immediately calls `self.cmc.settings.save()` from `on_radio_change` after assigning the selected value.

Required interactions:
- Initial render reads `update_source` and `log_level` from the loaded Rust settings model and selects the matching radio option before the tab is shown.
- Selecting any Update Channel option immediately requests persistence of the corresponding `update_source` value.
- Selecting any Log Level option immediately requests persistence of the corresponding `log_level` value.
- Radio groups are mutually exclusive within each group and independent across groups.
- There is no `Save`, `Apply`, `Cancel`, `Reset`, browse, or destructive confirmation control in Phase 2.
- Keyboard navigation must support moving focus between radio options and activating the focused option with the platform-standard key behavior supplied by Slint/native widgets.
- On save success, the selected option remains selected and no disruptive dialog appears.
- On save failure, revert the visible radio selection to the last persisted value and show the Error state copy in a non-modal status line within the Settings tab.

Out-of-scope interaction guardrails:
- Do not add scanner checkbox UI to Settings in this phase.
- Do not perform update checks, downloads, game discovery, scanner diagnostics, downgrader actions, or archive patching from these controls.
- Do not add settings that are absent from the reference app.

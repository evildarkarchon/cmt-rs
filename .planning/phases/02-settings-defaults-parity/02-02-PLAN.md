<interfaces>
Consume `crate::domain::settings::AppSettings` and its JSON repair/default APIs from Plan 01. Expose a small platform boundary so later app/controller code does not perform direct filesystem work.

Required store identifiers for downstream plans:
- `SettingsStore`: load/save service for `AppSettings`.
- `SettingsPaths` or equivalent: injectable settings path plus asset resolver root/path for tests.
- `AssetResolver` or equivalent trait/function: resolves `download-source.txt`; if the file is absent or invalid, update source falls back to `nexus` per D-04 and research pitfall 2.
</interfaces>
</context>

<decision_coverage>
- D-01: The production settings store must use current-directory `settings.json` as the default path.
- D-02: The settings store must accept injectable paths so tests use temp files and never touch real user or repository settings.
- D-03: Loading with no `settings.json` must create and save the default settings file.
- D-04: `download-source.txt` lookup must go through an asset resolver abstraction instead of assuming a filesystem location beside `settings.json` or compile-time embedding.
</decision_coverage>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem → `SettingsStore::load` | User-editable `settings.json` and `download-source.txt` cross into application state. |
| app/controller → `SettingsStore::save` | UI-selected values are persisted to local disk and may fail due to permissions, locks, or invalid paths. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05 | Tampering | `settings.json` load path | mitigate | Use Plan 01 per-key validation; reset malformed JSON; remove unknown keys on resave; test missing/malformed/partial cases. |
| T-02-06 | Tampering | `download-source.txt` asset resolver | mitigate | Accept only known update source wire values and fallback to `nexus` for missing or invalid content. |
| T-02-07 | Denial of Service | settings save | mitigate | Return IO errors to caller; never panic; tests cover save failure where feasible. |
| T-02-08 | Elevation of Privilege | path injection in tests/production | mitigate | Production path is fixed to current-directory `settings.json`; injected paths are test/controller explicit and not derived from untrusted UI input. |
</threat_model>

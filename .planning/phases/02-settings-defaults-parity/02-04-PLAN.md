<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user selection → Slint callback → controller | Untrusted UI event strings become persisted enum values. |
| controller → settings store | Application state crosses into filesystem persistence, which can fail. |
| settings store → Slint properties | Persisted state becomes visible UI state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-13 | Tampering | callback string values | mitigate | Parse only known `UpdateSource` and `LogLevel` wire values; reject or revert invalid strings in controller tests. |
| T-02-14 | Repudiation | save failure after radio selection | mitigate | Controller returns last persisted value on failure and logs/records the failure; UI property is reset to that value per D-07. |
| T-02-15 | Integrity | UI state vs persisted state | mitigate | MainWindow properties are initialized from loaded settings and updated only with controller-returned persisted/reverted values. |
| T-02-16 | Denial of Service | callback save path | mitigate | Save errors do not panic or block with modal loops; they return through controller and keep UI usable. |
</threat_model>

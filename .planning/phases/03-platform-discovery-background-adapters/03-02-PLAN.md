<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Filesystem paths/text -> adapters | User-controlled paths and file contents enter Rust through fakeable IO APIs. |
| Registry/process OS state -> typed values | OS registry/process data may be missing, malformed, or unavailable. |
| URL/path/tool target -> desktop launcher | External targets cross into OS handlers or process execution. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02-01 | D | `src/platform/filesystem.rs` | mitigate | Return `Result` errors for reads/enumeration; sort entries deterministically; do not panic on missing paths. |
| T-03-02-02 | I | `src/platform/registry.rs`, `src/platform/process.rs` | mitigate | Separate safe user-facing messages from raw diagnostics; return `UnsupportedPlatform` on non-Windows direct registry access per D-15. |
| T-03-02-03 | T/E | `src/platform/desktop.rs` | mitigate | Use typed operation kinds and target values; avoid shell-string construction; return typed failure values instead of dialogs/log-only failures. |
</threat_model>

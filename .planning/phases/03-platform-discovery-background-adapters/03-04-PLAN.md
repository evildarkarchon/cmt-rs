## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Worker thread -> UI event loop | Owned background results cross into Slint/UI state. |
| Long-running task closure -> event sink | Task errors/progress become user-visible events. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-04-01 | D | `src/workers/mod.rs` | mitigate | Run blocking work through worker facade/spawn_blocking and emit progress instead of blocking Slint callbacks. |
| T-03-04-02 | T | `src/workers/handoff.rs` | mitigate | Accept owned `WorkerEvent` values and apply through `slint::invoke_from_event_loop`; do not share mutable Slint models across threads. |
| T-03-04-03 | I | `src/workers/events.rs` | mitigate | Worker errors expose safe messages and keep diagnostics separate from user-facing payloads. |
</threat_model>
